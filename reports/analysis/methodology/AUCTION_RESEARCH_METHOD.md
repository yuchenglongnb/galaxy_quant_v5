# 竞价研究方法论

更新时间: 2026-06-16

## 目标

这份方法论文档用于统一 GalaxyQuant 的竞价分析、盘后验证、经验沉淀与后续升级方式。

原则不是把每天的市场表现硬编码成固定结论，而是把:

1. 稳定事实沉淀为规则或配置
2. 当日经验沉淀为结构化 lesson
3. 重复出现的市场模式升级为 pattern
4. 样本足够后再进入权重调整或模型排序

## 四层沉淀框架

### 1. 技能层

用途: 固定分析流程，保证以后每次复盘都按一致结构输出。

适合放入 skill 的内容:

- 数据状态检查
- 市场环境判断顺序
- 指数 / ETF / 个股 / 行业分层分析
- 验证统计口径
- 失败归因方式
- 复盘输出格式
- 经验沉淀与后续观察点

不适合放入 skill 的内容:

- 某一天的市场结论
- 还没有重复验证的临时经验

### 2. 规则层

用途: 存放明确、稳定、可重复验证的逻辑。

适合进入硬编码或配置的内容:

- CP/SA/趋势的触发条件
- 不同环境下的短名单门控
- 观察池与主执行区的区分
- 指数 / ETF / 个股的优先级
- 09:35 确认层是否可用时的可信度降级
- 除权、极端跳空、异常价格断点过滤

升级规则:

- 同类现象重复出现至少 3 次
- 不同日期下结论方向一致
- 可被明确数值条件表达

### 3. Lesson 层

用途: 记录当日案例、失败假设和修正建议。

lesson 是经验观察，不直接自动修改规则。

建议字段:

```json
{
  "date": "YYYYMMDD",
  "source": "auction_review",
  "signal_type": "CP风险|反核机会|趋势机会|market_env|other",
  "case": "...",
  "facts": {},
  "lesson": "...",
  "suggested_change": "...",
  "status": "observed|implemented|rejected"
}
```

升级路径:

```text
单日 lesson
-> 同类 lesson 重复出现
-> pattern
-> 规则/配置或排序权重
```

### 4. Pattern 层

用途: 记录已经具有复用价值的市场模式模板。

pattern 不是单日点评，而是可以跨日期调用的模式识别结果。

适合收录:

- risk_off 中的低开修复日
- 主线集中在某几个科技簇时的相对强弱判断
- CP 在主题强修复日的常见误判类型

pattern 应包含:

- 识别条件
- 主要证据字段
- 默认结论
- 默认动作建议
- 禁止误用条件

## 固定分析流程

后续竞价复盘默认按这个顺序执行:

1. 检查缓存状态: intraday / closed / missing
2. 检查 T-1 / T-2 是否完整
3. 判断指数环境
4. 判断 ETF 资金方向
5. 判断个股与行业的相对强弱
6. 输出短名单结构
7. 做验证统计
8. 找成功样本与失败样本
9. 提炼 lesson
10. 判断是否需要升级为 pattern 或规则

## 经验升级准则

### 直接进入规则

满足以下条件时可以考虑直接进入硬编码或配置:

- 逻辑是明确数值条件
- 多次复盘都稳定成立
- 错误成本高，提前过滤收益明显

例子:

- CP 文案必须区分 "高开诱多" 和 "强势后弱开兑现"
- 无 09:35 分时确认数据时，实时结论降级
- 个股 CP 默认先入观察池，不直接入主执行区

### 先进入 lesson

满足以下条件时先只记录为 lesson:

- 只在 1-2 天里出现
- 带明显市场语境
- 还不能稳定转成参数

例子:

- 某天反核整体明显优于 CP
- 某天指数不强但科技主线极强

### 从 lesson 升级为 pattern

满足以下条件时升级为 pattern:

- 至少 3 次出现
- 样本间具有相同结构
- 可以给出统一识别条件和默认动作

## 当前建议的几个重点模式

### Pattern A: repair_in_riskoff

含义:

- 指数普遍低开
- 收盘实体明显强于竞价
- 市场不是强趋势推进，而是低开承接修复

默认结论:

- CP 降权
- ETF / 指数反核升权
- 趋势只保留主线中的强个股

### Pattern B: cp_false_positive_in_theme_repair

含义:

- 个股前一日偏强
- 当日弱开或平开
- 按 CP 被判为兑现风险
- 但所属 ETF / 板块处于强修复

默认结论:

- 不应直接视为高开诱多
- 应描述为 "强势后弱开兑现待验证"
- 若主线强修复，需降低 CP 权重

### Pattern C: theme_cluster_repair

含义:

- 成功样本集中在同一科技链
- ETF、行业聚合、个股的修复方向一致

默认结论:

- 说明市场强度来自主线簇，而不是指数普涨
- 趋势与反核应优先服务于该主线簇

### Pattern D: strong_repair_continuation

含义:

- 前序市场连续承压，指数被 3 / 5 日线压制
- 当日竞价阶段指数与 ETF 整体显著转正
- `09:35 confirmed_strength` 向主线簇集中
- 收盘演化为科技成长或强主线的系统性修复

默认结论:

- 这不是普通 continuation，而是弱势后的强修复日
- 机械 CP 需要明显降权
- 允许 `09:35` 确认对 `09:25` 的保守判断做翻案
- 主线簇一致性应优先于单票竞价形态

### Pattern E: strong_repair_with_rotation

含义:

- 指数层面出现修复，尤其是上证或权重指数带动风险偏好回暖
- 但双创或旧强主线并没有形成全面趋势延续，反而高开后出现明显兑现
- 当日最强方向切换到新的主线簇，例如资源 / 有色 / 券商 / 部分软件
- 因而市场不是“全面做多日”，而是“结构性切换日”

默认结论:

- `strong_repair` 不等于 `trend` 全开
- 不应把双创高开直接解释成全面科技回流
- 旧高位科技股要优先防兑现
- 新强主线簇中的 ETF / 行业 / 中军更值得优先观察

默认动作:

- 将这类日子的交易语言固定为：
  - 做新强主线
  - 回避高位老主线追高
  - 允许少量低开承接
  - 不做泛趋势普开
- CP 需要加入“主线切换豁免”视角：
  - 若标的属于当日最强新簇，CP 不能直接当作否定结论
- 趋势需要增加三道过滤：
  - 相对所属 ETF 强弱
  - 相对指数强弱
  - 是否属于 leading cluster

禁止误用:

- 不把单日资源/券商走强简单等价为长期风格切换
- 不把 `strong_repair_with_rotation` 误判成 `broad_strong_repair`
- 不在没有盘后验证前把盘中观察直接升级为规则

### Pattern F: shrinking_volume_trend_continuation

含义:

- 指数层面不是强放量突破，而是前一日修复后进入缩量延续
- 四大指数多数没有明显转弱，成长指数和主线 ETF 仍维持正实体
- leading cluster 没有瓦解，趋势延续主要由主线簇内部中军、强个股和核心 ETF 支撑
- 高 CP 不再天然等于兑现，低开承接和主线簇一致性比“反向猜顶”更重要

默认结论:

- 这不是全面普涨日，而是“可以做结构，不适合做普涨”的缩量趋势延续日
- 可以跟主线，但不适合单纯依赖 CP 去反向猜顶
- 低开承接优先级高于高开兑现判断

默认动作:

- CP 增加 `leading_cluster` 豁免：
  - 若标的属于当日主线簇，且 ETF / 指数相对强弱未转弱，则 CP 只作为拥挤观察，不直接否定
- 趋势候选继续压缩：
  - 相对所属 ETF 强弱
  - 相对指数强弱
  - 主线簇强度 / leading cluster 一致性
- 反核只保留主线低开承接，不做弱板块普反

禁止误用:

- 不把缩量延续误判成普涨牛市
- 不把所有高 CP 样本都解释成兑现风险
- 不在主线簇仍强时，对核心方向机械做逆向

  - structural_env：主线结构环境

- structural_env 负责回答“能不能围绕主线集中做多”

## 后续维护方式

每次出现新的明显市场情况时:

1. 先在 `reports/analysis/lessons/auction_lessons.jsonl` 记 lesson
2. 判断是否属于已有 pattern
3. 如不属于，补充到 `reports/analysis/patterns/market_pattern_registry.json`
4. 如重复出现，再决定是否改规则或权重

## 盘中分析与盘后验证闭环

后续若在盘中使用数据做判断，默认按以下闭环执行:

1. 先输出盘中读取:
   - 当前环境
   - 当前主线簇
   - 当前可执行方向
2. 同时输出待验证假设:
   - 哪些指数 / ETF / 个股必须保持强势到收盘
   - 哪些旧强方向若继续兑现，则说明盘中判断成立
3. 收盘后用 `signal_metrics.csv` 和 `signal_detail.csv` 回测盘中判断
4. 若盘中判断与收盘验证不一致:
   - 先记 lesson
   - 再判断是否属于已有 pattern
   - 不直接把单日经验写成规则

## 注意事项

- 不把单日成功率直接等同于长期有效性
- 不把 AI 文案当作事实来源
- 不把 pattern 当成自动交易指令
- 任何升级都必须以验证数据为基础

## 环境优先原则

后续优化默认遵循“指数环境第一位，板块主线第二位，个股信号第三位”：

1. 先判断指数环境是否允许做多。
2. 再判断是否存在能够承接的主线簇。
3. 最后才讨论 CP / 反核 / 趋势在个股层面的可执行性。

当市场出现以下特征时，应优先视为“环境恶劣”或“弱势延续”：

- 四大指数多数低开后继续走弱
- 创业板 / 科创50 实体明显为负
- 指数持续受 3 日 / 5 日均线压制
- ETF 反核样本大面积失败
- 趋势样本虽然有个别成功，但整体平均实体仍为负

在这种环境里：

- 反核不再默认视为抄底机会
- 趋势信号需要大幅降权
- CP 更适合作为回避追高和兑现风险提示
- 主报告应优先输出“环境恶劣，谨慎或空仓观望”

只有当指数环境转向修复、主线簇内部承接重新稳定后，反核和趋势才恢复参考价值。
