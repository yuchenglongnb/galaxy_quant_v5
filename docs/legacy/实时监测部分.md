# 盘中监测功能使用指南

## 一、功能概述

盘中监测模块实现了 **9:25-15:00** 期间对四大指数和主题ETF的实时数据采集。

### 监测标的

| 类型 | 数量 | 说明 |
|------|------|------|
| 指数 | 4只 | 上证、创业板、科创50、北证50 |
| ETF | 27只 | 覆盖科技、资源、消费、金融、制造等板块 |

### 采集策略

| 时段 | 间隔 | 说明 |
|------|------|------|
| 9:25-9:30 | 60秒 | 竞价结束，可同步竞价数据 |
| 9:30-11:30 | 60秒 | 上午连续竞价 |
| 11:30-13:00 | 300秒 | 午休，降低频率 |
| 13:00-15:00 | 60秒 | 下午连续竞价 |

## 二、快速开始

### 2.1 查看市场状态

```bash
python main.py status
```

输出示例：
```
当前时间: 2025-01-21 10:30:00
市场状态: 上午连续竞价
状态: 🟢 可采集

📅 今日采集计划:
----------------------------------------
  09:25-09:30  竞价结束      每分钟采集
  09:30-11:30  上午连续竞价  每分钟采集
  11:30-13:00  午间休市      每5分钟采集
  13:00-14:57  下午连续竞价  每分钟采集
  14:57-15:00  收盘集合竞价  每分钟采集
```

### 2.2 启动盘中监测

**基础模式**（仅采集数据）：
```bash
python main.py monitor
```

**带市场概览**（每5分钟显示涨跌榜）：
```bash
python main.py monitor --summary
```

**测试模式**（采集一次后退出）：
```bash
python main.py monitor --test
```

### 2.3 后台运行（推荐）

使用 `nohup` 或 `tmux` 保持程序运行：

```bash
# 方式1: nohup
nohup python main.py monitor --summary > monitor.log 2>&1 &

# 方式2: tmux
tmux new -s monitor
python main.py monitor --summary
# Ctrl+B D 分离会话
```

|------|------|------|
| code | str | 证券代码 |

| time_int | int | 时间 (HHMM) |
| time_str | str | 时间 (HH:MM:SS) |
| pre_close | float | 昨收价 |

| high | float | 最高价 |
| low | float | 最低价 |
| last | float | 最新价 |
| volume | int | 累计成交量 |
| amount | float | 累计成交额 |
| volume_1min | int | 分钟成交量（增量） |
| amount_1min | float | 分钟成交额（增量） |
| pct | float | 涨跌幅 (%) |

| 参数 | 说明 |
|------|------|
| --summary | 显示市场概览（每5分钟） |

## 六、断点续传

程序支持断点续传。如果程序中断后重新启动，会自动检测上次采集的时间点并继续：

```

   将从断点继续采集...
```

## 九、配置修改

### 9.1 添加ETF

编辑 `config/settings.py`：

```python
class MarketConfig:
    THEME_ETFS = {
        # ... 已有ETF
        "新代码.SH": "新名称",  # 添加新ETF
    }
```

### 9.2 修改采集间隔

编辑 `config/settings.py`：

```python
class MonitorConfig:
    DEFAULT_INTERVAL = 30        # 改为30秒
    NOON_INTERVAL_MULT = 10      # 午休10分钟一次
```

## 十、常见问题

### Q1: 程序报错 "已收盘，无法启动监测"

**原因**：在非交易时间运行

**解决**：请在交易时间（9:25-15:00）运行，或使用 `--test` 模式测试

### Q2: 数据采集不完整

**原因**：API网络问题

**解决**：程序会自动重试，如果持续失败请检查网络和API凭证

### Q3: 午休期间仍然每分钟采集

**原因**：默认午休降频到5分钟

**解决**：这是正常行为，可以通过配置修改
